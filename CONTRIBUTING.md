# Contributing to TransCore

Спасибо за интерес к проекту. TransCore — открытое локальное ядро
перевода, и главная ценность проекта — приватность и независимость
от внешних сервисов. Любой вклад должен сохранять этот принцип.

## Основные правила

1. **Никаких внешних API и облачных сервисов.**
   Любой модуль должен работать полностью офлайн. PR с интеграцией
   Google/DeepL/OpenAI и подобных API в ядро — отклоняются.

2. **Ядро (`core/`) не трогаем без обсуждения.**
   `TranslationContext`, `TranslationEngine`, `Module` — стабильный
   фундамент. Новую функциональность добавляй через модули (`modules/`),
   а не через изменение самого движка.

3. **Каждый модуль — отдельный класс с методом `process(ctx)`.**
   Никаких скрытых побочных эффектов и обращений к диску/сети,
   кроме явной загрузки локальных данных (словарей и т.п.).

4. **Тесты обязательны.**
   Новый модуль без теста в `tests/` не принимается. Тест должен
   покрывать хотя бы базовый и один контекстный случай.

5. **Данные — только открытые источники.**
   Словари/корпуса берём из открытых баз (OPUS, Wiktionary, Panlex
   и т.п.), с указанием источника в PR.

## Как добавить новый язык

1. Добавь `data/dict_<lang1>_<lang2>.json` с той же структурой,
   что и `dict_en_ru.json`.
2. Если язык требует особой обработки (другой порядок слов,
   агглютинация и т.п.) — обсуди в Issues перед PR, возможно
   потребуется доп. модуль в пайплайне.
3. Добавь тест с реальными примерами перевода.

## Как отправить PR

1. Форкни репозиторий
2. Создай ветку `feature/<короткое-описание>`
3. Убедись что все тесты проходят (`pytest tests/`)
4. Опиши в PR что именно меняется и зачем

## Стиль кода

- Python 3.10+
- Названия классов — CamelCase, функций/переменных — snake_case
- Комментарии — по-русски или по-английски, главное понятно

Лицензия проекта — Apache 2.0. Любой вклад распространяется на тех
же условиях.
